# Portfolio Checklist

What to include in your professional portfolio.

## Overview

Your portfolio demonstrates your skills to potential employers, clients, and collaborators. This checklist helps you present your learning OS projects professionally.

## Portfolio Structure

### Essential Sections

1. **About** - Who you are, what you do
2. **Projects** - Your best work (3-5 featured)
3. **Skills** - Technologies you know
4. **Experience** - Work history (if applicable)
5. **Contact** - How to reach you

### Per-Project Information

Each project should include:

| Element | Description |
|---------|-------------|
| Title | Clear, descriptive name |
| Summary | 1-2 sentence overview |
| Problem | What you solved |
| Solution | How you solved it |
| Technologies | Stack used |
| Demo | Video/GIF/live link |
| Code | GitHub link |
| Write-up | Blog post (optional) |

## Project Documentation Checklist

For each month's project:

### README.md

- [ ] Clear title and description
- [ ] Problem statement
- [ ] Solution overview
- [ ] Installation instructions
- [ ] Usage examples
- [ ] Demo GIF/video
- [ ] Technologies listed
- [ ] License

### Code Quality

- [ ] Clean, readable code
- [ ] Consistent style (ruff/black)
- [ ] Comments where needed
- [ ] Type hints (Python)
- [ ] Tests included

### Demo Assets

- [ ] Demo video (3-5 min)
- [ ] Demo GIF (30s-1min)
- [ ] Screenshots
- [ ] Sample outputs

### Documentation

- [ ] API documentation (if applicable)
- [ ] Architecture diagram
- [ ] Setup guide

## Selecting Projects

Choose projects that show:

### Variety

| Type | Examples |
|------|----------|
| Data/ML | Model training, evaluation |
| Backend | APIs, data pipelines |
| Full-stack | End-to-end applications |
| Infrastructure | Deployment, monitoring |

### Depth

Include at least one project that shows:
- Complex problem-solving
- Multiple technologies working together
- Production considerations
- Iteration based on feedback

### Recency

Prioritize recent work:
- Last 6-12 months preferred
- Update older projects if still relevant
- Remove outdated technologies

## Platform Options

### GitHub

**Essential**: Your code lives here.

- Clean repositories
- Good READMEs
- Pinned repositories for best work
- Green contribution graph

### Personal Website

**Recommended**: Professional presentation.

Options:
- GitHub Pages (free)
- Vercel/Netlify (free tier)
- Custom domain ($12/year)

### LinkedIn

**Important**: Professional network.

- Projects section
- Featured content
- Skills endorsements

### Medium/Blog

**Good to have**: Shows communication skills.

- Technical write-ups
- Lessons learned
- Tutorials

## Month-by-Month Portfolio Building

| Month | Portfolio Action |
|-------|------------------|
| 1-2 | Set up portfolio site |
| 3-4 | Add first 2 projects |
| 5-6 | Add projects 3-4, refine earlier entries |
| 7-8 | Add projects 5-6, update skills |
| 9-10 | Polish presentation, add demos |
| 11-12 | Final projects, comprehensive review |

## Quality Checks

### Before Adding to Portfolio

- [ ] Project works (tested recently)
- [ ] README is complete
- [ ] No secrets in code
- [ ] Demo assets created
- [ ] Links all work
- [ ] Professional presentation

### Quarterly Review

Every 3 months:
- [ ] Remove outdated projects
- [ ] Update descriptions
- [ ] Refresh demo videos
- [ ] Add new skills
- [ ] Check all links

## Common Mistakes

**Avoid these**:

| Mistake | Fix |
|---------|-----|
| Too many projects | Curate to 3-5 best |
| No demos | Add video/GIF for each |
| Broken links | Test regularly |
| Generic descriptions | Be specific about your role |
| Outdated tech | Note if legacy, or update |
| No README | Every repo needs one |

## Example Portfolio Sites

When building your portfolio, study examples:
- Clean, minimal design
- Easy navigation
- Fast loading
- Mobile-friendly
- Accessible

## Technical Portfolio Tips

### For ML/AI Projects

Show:
- Model metrics (accuracy, F1, etc.)
- Evaluation methodology
- Data handling
- Production considerations

### For Backend Projects

Show:
- API documentation
- Performance metrics
- Scalability considerations
- Testing approach

### For Full-Stack Projects

Show:
- End-to-end flow
- User experience
- Backend architecture
- Deployment setup

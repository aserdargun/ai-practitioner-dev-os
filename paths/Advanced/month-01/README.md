## Objectives

By the end of this month, you will:

1. Have a fully configured development environment
2. Understand Git workflows for ML projects
3. Be comfortable with Python best practices
4. Have your first working project committed

---

## Weekly Breakdown

### Week 1: Development Environment

**Focus**: Set up everything you need

**Tasks**:
- [ ] Install Python 3.11+
- [ ] Set up VS Code with extensions
- [ ] Configure Git and SSH keys
- [ ] Create virtual environment
- [ ] Install core packages (pandas, numpy, pytest, ruff)

**Deliverables**:
- Working Python environment
- VS Code configured with Python extension
- Git configured with SSH

### Week 2: Python Best Practices

**Focus**: Write clean, maintainable Python

**Tasks**:
- [ ] Review PEP 8 style guide
- [ ] Set up ruff for linting
- [ ] Learn type hints
- [ ] Practice with Pydantic models
- [ ] Write your first test with pytest

**Deliverables**:
- Linted Python project
- Tests passing
- Type-annotated functions

### Week 3: Git Workflows

**Focus**: Master version control for ML

**Tasks**:
- [ ] Learn branching strategies
- [ ] Practice commit message conventions
- [ ] Understand .gitignore for ML projects
- [ ] Learn to handle large files (Git LFS concepts)
- [ ] Set up pre-commit hooks

**Deliverables**:
- Feature branch workflow
- Clean commit history
- Proper .gitignore

### Week 4: First Project

**Focus**: Build something small but complete

**Tasks**:
- [ ] Choose a small project (data processing script)
- [ ] Apply all learned practices
- [ ] Write tests
- [ ] Document with README
- [ ] Ship it!

**Deliverables**:
- Complete mini-project
- README with usage instructions
- Tests passing
